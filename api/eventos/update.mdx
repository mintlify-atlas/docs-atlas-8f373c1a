## PUT /api/eventos/:id

Updates an existing event and returns the updated `Evento` object. You can send any subset of the event fields — only the provided fields are modified.

<Warning>
  This endpoint requires a valid admin Bearer token in the `Authorization` header.
</Warning>

## Request

**Path parameters**

<ParamField path="id" type="number" required>
  The `id_evento` of the event to update.
</ParamField>

**Headers**

<ParamField path="header.Authorization" type="string" required>
  Bearer token for admin authentication. Format: `Bearer <token>`.
</ParamField>

<ParamField path="header.Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

**Body**

All body fields are optional. Include only the fields you want to change.

<ParamField body="nombre" type="string">
  Updated display name of the event.
</ParamField>

<ParamField body="descripcion" type="string">
  Updated description of the event.
</ParamField>

<ParamField body="fecha_inicio" type="string">
  Updated start date in ISO 8601 format.
</ParamField>

<ParamField body="fecha_fin" type="string">
  Updated end date in ISO 8601 format.
</ParamField>

<ParamField body="id_edificio" type="number">
  Updated building ID.
</ParamField>

<ParamField body="publico" type="boolean">
  Updated public visibility flag.
</ParamField>

<ParamField body="activo" type="boolean">
  Updated active status.
</ParamField>

## Response

Returns the updated `Evento` object. See [List events](/api/eventos/list) for the full field reference.

## Example

```bash
curl -X PUT http://localhost:3001/api/eventos/12 \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Feria de Ciencias 2026",
    "activo": false
  }'
```

```json
{
  "id_evento": 12,
  "nombre": "Feria de Ciencias 2026",
  "descripcion": "Exposición de proyectos estudiantiles.",
  "fecha_inicio": "2026-05-15T09:00:00.000Z",
  "fecha_fin": "2026-05-15T17:00:00.000Z",
  "id_edificio": 2,
  "publico": true,
  "activo": false,
  "edificio": {
    "id_edificio": 2,
    "nombre": "Facultad de Ingeniería",
    "tipo": "academico",
    "latitud": 19.4321,
    "longitud": -99.1340
  }
}
```

## Error codes

| Status | Description |
|--------|-------------|
| `400`  | Invalid field values in the request body. |
| `401`  | Missing or invalid Bearer token. |
| `404`  | No event found with the given `id`. |
| `500`  | Internal server error. |
