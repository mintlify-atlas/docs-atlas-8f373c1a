AirGuide is split into two packages: a React/TypeScript frontend at the project root and a Node.js/Express backend in the `server/` directory. Both share a single PostgreSQL database managed through Prisma Cloud.

### Authentication

JWT-based authentication using `jsonwebtoken`. The API expects a `Bearer <token>` header on protected routes. Tokens are signed with `JWT_SECRET` and expire after the duration set in `JWT_EXPIRES_IN`.

The frontend stores the token in `localStorage` and attaches it to mutating requests:

```typescript
const token = localStorage.getItem('token');
const response = await fetch(`${API_URL}/edificios`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`,
  },
  body: JSON.stringify(data),
});
```

### Database

PostgreSQL hosted on **Prisma Cloud**. The connection string in `DATABASE_URL` includes connection pooling (`pool=true`) and SSL (`sslmode=require`).

Prisma Studio provides a visual interface to browse and edit records:

```bash
cd server
npm run prisma:studio
```

## Design system

The design system is defined in `design-tokens.json` and exposed as CSS custom properties. All component styles reference these variables rather than hard-coded values.
