The map loads at `/` for all visitors — no login required. Buildings and events are fetched automatically on page load and rendered as color-coded Leaflet markers.

## Map initialization

The map is initialized with the Leaflet `L` API directly (not react-leaflet). It centers on the university campus at coordinates `[20.656333, -100.404745]` with zoom level `15`.

```tsx Map.tsx
const map = L.map(mapContainerRef.current).setView([20.656333, -100.404745], 15);

L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
    attribution: '&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors'
}).addTo(map);
```

Tiles are served by OpenStreetMap. The map container takes up the full remaining viewport height below the header and search bar.

## Search bar

The search bar sits below the top header and filters visible markers in real time. It matches against both `nombre` and `descripcion` fields for buildings and events.

```tsx Map.tsx
const edificiosFiltrados = edificios.filter(e =>
    e.nombre.toLowerCase().includes(searchTerm.toLowerCase()) ||
    e.descripcion?.toLowerCase().includes(searchTerm.toLowerCase())
);

const eventosFiltrados = eventos.filter(e =>
    e.nombre.toLowerCase().includes(searchTerm.toLowerCase()) ||
    e.descripcion?.toLowerCase().includes(searchTerm.toLowerCase())
);
```

When the search term changes, the marker layer is cleared and rebuilt with only the matching results. Clearing the search field restores all markers.

## Marker info panel

Clicking any marker opens a slide-in info panel in the top-right corner of the map. The panel shows different content depending on whether the marker is a building or an event.

**For buildings:**

- Name and a `tipo` badge (academico, administrativo, or recreativo)
- Description (if present)
- A 3-column counts grid showing the number of salones, cubículos, and eventos

**For events:**

- Name and a green calendar badge
- Description (if present)
- Start and end dates
- Building name (the location where the event is held)

<Note>
  The panel includes a "Cómo llegar" (How to get there) button that is only visible to users with the `alumno` role. Guests see a prompt to log in for more route options.
</Note>

## Authentication-aware header

The top header adapts based on the current authentication state.

<CardGroup cols={2}>
  <Card title="Guest" icon="user">

  </Card>
  <Card title="Logged in" icon="shield-halved">
    Shows the user's name and role badge, a **Cerrar Sesión** button, and — for admins — a **Dashboard** link to `/admin`.
  </Card>
</CardGroup>

The user session is read from `AuthContext`, which hydrates from `localStorage` on page load.

## Map legend

A small legend panel is fixed to the bottom-left corner of the map. It identifies the two marker types:

- Building icon (blue) — Edificios
- Calendar icon (green) — Eventos

## Loading state

While building or event data is being fetched, a centered overlay with a spinning indicator and the label "Cargando mapa..." covers the map. The overlay is dismissed once both requests resolve.
