The search bar lets you narrow down what appears on the map. As you type, AirGuide hides markers that don't match and keeps the ones that do — without reloading the page.

## Using the search bar

The search input sits in the header bar directly below the main navigation, spanning the full width of the page (max 2xl). It is always visible on the map page.

Type any word or phrase into the bar. The map updates immediately — no need to press Enter.

To clear the search and restore all markers, delete the text from the input.

## What gets searched

Every keystroke filters **both buildings and events at the same time**:

| Data type | Fields searched |
|---|---|
| Buildings | `nombre` (name), `descripcion` (description) |
| Events | `nombre` (name), `descripcion` (description) |

The match is case-insensitive. Typing `AULA`, `aula`, or `Aula` all return the same results.

## How filtering works

The filter runs entirely on the client using data already loaded in memory. Here is the exact logic from `Map.tsx`:

```typescript
// Filtrar edificios y eventos según búsqueda
const edificiosFiltrados = edificios.filter(e =>
    e.nombre.toLowerCase().includes(searchTerm.toLowerCase()) ||
    e.descripcion?.toLowerCase().includes(searchTerm.toLowerCase())
);

const eventosFiltrados = eventos.filter(e =>
    e.nombre.toLowerCase().includes(searchTerm.toLowerCase()) ||
    e.descripcion?.toLowerCase().includes(searchTerm.toLowerCase())
);
```

Whenever `searchTerm` changes, the filtered arrays are recomputed and the map markers are fully replaced — non-matching markers are removed, matching markers remain.

<Note>
  Event markers are placed at the coordinates of the building they belong to. If a building is filtered out but one of its events matches your search, the event marker will still appear at that building's location.
</Note>
